# Auto-QA Dashboard: 경쟁사 기능 비교 및 1단계 경쟁력 검증 리포트

## 1. 검증 목적 및 Phase 1 경쟁력
현재 Phase 2(RAG 아키텍처) 도입 전임에도 불구하고, Phase 1의 '범용 LLM 기반 5가지 원인 자동 분류'와 '도메인 특화 웹 UI'만으로 기존 글로벌 데이터 품질 관리 도구를 압도하는 현업(패널 관리자) 실무 투입성(Time-to-Value)을 증명합니다.

## 2. 핵심 경쟁 도구 기능 비교 검증
| 비교 항목 | Auto-QA Dashboard (Phase 1) | Great Expectations | Soda Core |
| :--- | :--- | :--- | :--- |
| **핵심 검수 방식** | Rule-based 검수 + AI 원인 추론 | 순수 Rule-based (Expectations) | 순수 Rule-based (YAML checks) |
| **오류 원인 분석** | 5가지 카테고리 (정책/누락 등) 자동 분류 | 불가능 (T/F 반환) | 불가능 (임계치 이탈 경고만) |
| **예외 상황 처리** | AI 문맥 분석으로 정책 변경/누락 구분 | 임계치 이탈 시 무조건 오류 차단 | 복잡한 커스텀 코드 작성 필수 |
| **사용자 인터페이스** | No-Code 웹 UI 대시보드 | Python 코드 작성 및 CLI 필수 | YAML/SQL 설정 및 CLI 필수 |
| **실무자 편의성** | Excel 다운로드 및 보고서 자동 생성 | HTML 정적 문서 (Data Docs) | 커스텀 리포팅 기능 미흡 |

## 3. 핵심 검증 시나리오 (Rule-based 한계 극복)
* **시나리오 A (정책 변경 vs 누락 구분):** 특정 제약사 단가가 10% 인상되었을 때, 기존 도구는 무조건 '오류(Anomaly)'로 차단함. Auto-QA는 LLM 문맥 분석을 통해 "일괄적인 단가 변동 패턴(정책 변경)"으로 정확히 자동 분류하여 수기 확인 시간을 100% 제거함.
* **시나리오 B (Time-to-Value):** 코딩 지식이 없는 패널 관리자가 Python 스크립트 없이 UI 파일 업로드만으로 검수부터 보고서(Excel) 출력까지 완료하여 기존 2~3일 소요되던 업무를 80% 단축함.

## 4. 로드맵 연계를 위한 데이터 에셋화 전략 (Data Assetization)
현재 Phase 1에서 스마트 캐싱을 통해 누적되는 '제약 데이터 오류 패턴', 'AI 원인 분류 결과', '실무자 최종 승인 로그'는 Phase 2의 RAG 벡터 DB 구축을 위한 가장 완벽하고 정제된 기초 데이터 에셋(Asset)으로 활용됩니다.
