## 시나리오 1 — 표준 컬럼 파일 검수 (Happy Path)

### 상황
월말 처방 데이터 파일(전월: `2025-11.csv`, 당월: `2025-12.csv`)이 준비됨.
파일 컬럼: `drug_id`, `drug_name`, `hospital_code`, `prescription_volume`, `date`

### 단계별 흐름

```
[1] 대시보드 접속
    └─> https://helloworld-3hhd.vercel.app/dashboard

[2] 파일 업로드
    ├─> 좌측 드롭존: 전월 파일(2025-11.csv) 드래그앤드롭
    ├─> 우측 드롭존: 당월 파일(2025-12.csv) 드래그앤드롭
    └─> "다음: 컬럼 매핑" 버튼 클릭

[3] 컬럼 매핑
    ├─> 파일 헤더가 시스템 필드와 일치하므로 드롭다운 자동 인식
    └─> "검수 시작" 클릭

[4] 검수 결과 확인 (약 1초)
    ├─> 요약 카드: 총 1,243건 / 정상 1,180건 / 경고 48건 / 위험 15건
    ├─> 기준 기간: 2025-11 / 검수 기간: 2025-12 표시
    └─> 이상치 테이블: 변동률 내림차순 정렬

[5] AI 원인 분석 (선택)
    ├─> "AI 분석 실행" 클릭
    ├─> 경고/위험 63건에 대해 Claude AI 배치 분석 (~15초)
    └─> 분류 요약: data_error 8건 / market_trend 21건 / seasonal 12건 / ...

[6] 보고서 다운로드
    ├─> "Excel 다운로드" 클릭
    └─> AI 분석 결과 포함 보고서 저장
```

### 기대 결과
- 검수 소요 시간: 수동 3~4시간 → **2분 이내**
- 담당자는 `data_error` 8건만 집중 확인하면 됨

## 시나리오 2 — 한글 컬럼명 파일 검수

### 상황
레거시 시스템에서 추출된 파일의 컬럼명이 한글: `품목코드`, `품목명`, `요양기관번호`, `처방량`, `기준년월`

### 단계별 흐름

```
[1~2] 파일 업로드까지 동일

[3] 컬럼 매핑 화면
    ├─> 약품 코드 (필수) → [품목코드 ▼] 선택
    ├─> 약품명           → [품목명 ▼] 선택
    ├─> 병원 코드 (필수) → [요양기관번호 ▼] 선택
    ├─> 처방량 (필수)    → [처방량 ▼] 선택
    ├─> 날짜             → [기준년월 ▼] 선택
    └─> "검수 시작" 클릭 (필수 3개 선택 완료 시 활성화)

[4~6] 이후 동일
```

## 시나리오 3 — AI 크레딧 부족 상황

### 상황
Anthropic 계정의 크레딧이 소진된 상태에서 AI 분석 실행

### 흐름

```
[5] AI 분석 실행 클릭
    ├─> API 오류 발생 (크레딧 부족)
    ├─> 시스템이 자동으로 규칙 기반 예시 분석 생성
    └─> 결과 표시:
        ├─> [예시 데이터] 배지 표시
        ├─> "AI API 연결 실패로 예시 분석 결과를 표시합니다" 안내
        └─> 규칙 기반으로 추정된 원인 분류 결과 표시
```

### 기대 결과
- 에러 화면 대신 예시 결과로 데모/발표 시 서비스 연속성 유지

---

## 시나리오 4 — 이상치 심층 분석

### 상황
AI 분석 완료 후 `data_error`로 분류된 항목 집중 검토

### 흐름

```
[1] 이상치 테이블에서 심각도 필터 "위험" 선택
[2] AI 분류 필터 "data_error" 적용 (향후 기능)
[3] 해당 약품/병원 코드 확인
[4] Excel 보고서 다운로드 → 원본 데이터와 대조
[5] 데이터 수정 후 재업로드하여 재검수
```

---

## UX 원칙

- **단계별 진행**: 업로드 → 매핑 → 결과의 명확한 3단계로 인지 부하 감소
- **즉각적 피드백**: 파일 선택 즉시 파일명/크기 표시, 검수 완료 즉시 결과 렌더링
- **Graceful Degradation**: AI 분석 실패해도 규칙 검수 결과는 항상 제공
